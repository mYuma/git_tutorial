GIT

/* 概要 */
Gitの８割はローカルリポジトリの操作
ワークツリー　→　ステージ　→　リポジトリ

ステージは１つのファイルを持つ
・インデックスファイル　圧縮ファイルとファイル名の対応表

リポジトリは３つのファイルを持つ
・圧縮ファイル　ファイルの内容を圧縮したもの
・ツリーファイル　インデックスのコピー
・コミットファイル　ツリーファイル、親コミット、日付とか

/* コマンド */
// Gitプロジェクトを作成する 作成したいフォルダで実行 Intialize
git init


// GitHub上にあるリポジトリからスタート
git clone <Repositly Name>
ex git clone http://


// 変更をステージに追加する
git add <ファイル名>
git add <ディレクトリ名>
→　一部のファイル・ディレクトリのみコミットしたいとき
　　(反映させたくないファイルがあるとか)

git add .
→　すべてのファイルをステージへ


// ステータスの確認
git status


// 変更差分の確認
git diff
git diff <FileName>

git diff --staged // addしたあとのファイルを見る場合
