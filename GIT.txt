GIT

/* 概要 */
Gitの８割はローカルリポジトリの操作
ワークツリー　→　ステージ　→　リポジトリ

ステージは１つのファイルを持つ
・インデックスファイル　圧縮ファイルとファイル名の対応表

リポジトリは３つのファイルを持つ
・圧縮ファイル　ファイルの内容を圧縮したもの
・ツリーファイル　インデックスのコピー
・コミットファイル　ツリーファイル、親コミット、日付とか

/* コマンド */
// Gitプロジェクトを作成する 作成したいフォルダで実行 Intialize
git init


// GitHub上にあるリポジトリからスタート
git clone <Repositly Name>
ex git clone http://


// 変更をステージに追加する
git add <ファイル名>
git add <ディレクトリ名>
→　一部のファイル・ディレクトリのみコミットしたいとき
　　(反映させたくないファイルがあるとか)

git add .
→　すべてのファイルをステージへ


// ステータスの確認
git status


// 変更差分の確認
git diff
git diff <FileName>

git diff --staged // addしたあとのファイルを見る場合


// 変更履歴の確認
git log

--oneline // １行で表示
-p <FileName>　// ファイルを指定して表示
-n <CommitCnt> // コミット数を指定して表示


// ファイルの削除
git rm <ファイル名>
git rm -r <ディレクトリ名>
→　ローカルのファイルも削除

git rm --cached <ファイル名>
→　ローカルには残す、gitから削除


// ファイルの移動
git mv <OldFileName> <NewFileName> // Stageまで反映される


// リモートリポジトリにPushする
git push <RemoteName/origin> <BranchName>

/* リモートリポジトリを新規追加する */
git remote add <RemoteName/origin> <URL>

/* リモートリポジトリのURLを確認する */
git remote -v

/* リモートリポジトリのURLを変更する */
git remote set-url <Remote> <NewURL>

/* 管理しないファイルをgitの管理から外す */
.gitignoreに指定する
